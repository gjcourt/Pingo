# Pull Request Workflow

- All changes must be made in a feature branch.
- Branch names should follow the format `feature/description`, `bugfix/description`, or `chore/description`.
- Create a Pull Request (PR) against the `main` branch.
- Ensure all tests and linters pass before requesting a review.
- Use the provided Pull Request template.
- Keep PRs small and focused on a single task.
- Write clear and descriptive commit messages.
